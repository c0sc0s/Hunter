# Agent LLM

## Goal

Hunter's first agent slice adds optional content understanding on top of saved items. It does not change source recognition, extraction state, or deterministic Content Signals.

The default runtime is local Ollama because it gives the desktop prototype a simple offline/local model path without adding a bundled native model runtime yet. The same agent boundary also supports OpenAI-compatible Chat Completions providers, including DeepSeek. A fully embedded on-device model remains a separate product/runtime track.

## Boundaries

- Recognition stays deterministic and snapshot-only.
- Agent classification reads only fields already captured by Hunter: title, URL, source metadata, summary, excerpt, readable text, sanitized content HTML text, note, and stored snapshot text.
- Agent results are stored on the saved item as `agentClassification` and displayed in the item detail panel.
- Agent content categories are user-facing filter buckets generated by the model. The model must reuse an existing category when it clearly fits and create a new concise category only when none fit.
- Missing runtime, missing API key, missing model, timeouts, invalid JSON, or schema mismatches return explicit failures. The API must not fake classification success.

## Runtime

Default provider: Ollama HTTP API.

The sidebar Settings modal is the preferred configuration surface. It stores provider, base URL, model, and API key under Hunter's local data directory. API keys are never returned to the renderer; the UI only sees `apiKeyConfigured`.

Environment variables remain available as defaults/fallbacks:

- `HUNTER_LLM_PROVIDER`: `ollama`, `deepseek`, or `openai-compatible`. Defaults to `ollama`.
- `HUNTER_LOCAL_LLM_BASE_URL`: Ollama base URL. Defaults to `http://127.0.0.1:11434`.
- `HUNTER_LOCAL_LLM_MODEL`: model name. Defaults to `llama3.2:3b`.
- `HUNTER_LOCAL_LLM_TIMEOUT_MS`: generation timeout. Defaults to `60000`.
- `HUNTER_LOCAL_LLM_TEMPERATURE`: generation temperature. Defaults to `0.1`.
- `HUNTER_DEEPSEEK_API_KEY` or `DEEPSEEK_API_KEY`: DeepSeek bearer token.
- `HUNTER_DEEPSEEK_BASE_URL`: DeepSeek base URL. Defaults to `https://api.deepseek.com`.
- `HUNTER_DEEPSEEK_MODEL`: DeepSeek model. Defaults to `deepseek-v4-flash`.
- `HUNTER_OPENAI_COMPATIBLE_BASE_URL`: generic Chat Completions-compatible base URL.
- `HUNTER_OPENAI_COMPATIBLE_API_KEY` or `OPENAI_API_KEY`: generic provider bearer token.
- `HUNTER_OPENAI_COMPATIBLE_MODEL`: generic provider model.

Ollama setup example:

```sh
ollama pull llama3.2:3b
HUNTER_LOCAL_LLM_MODEL=llama3.2:3b pnpm dev
```

DeepSeek setup example:

```sh
export HUNTER_LLM_PROVIDER=deepseek
export DEEPSEEK_API_KEY=<your key>
export HUNTER_DEEPSEEK_MODEL=deepseek-v4-flash
pnpm dev
```

## API

`GET /api/agent/local-llm`

Returns provider reachability, selected model, available models when the provider exposes them, whether the selected model exists, and API-key configuration status for API-backed providers.

`GET /api/agent/settings`

Returns the saved provider configuration without the raw API key.

`PATCH /api/agent/settings`

Stores provider configuration. An empty `apiKey` keeps the existing key for the same provider; switching providers without a replacement key clears the previous provider's key. `clearApiKey=true` removes the current key.

`POST /api/agent/items/classify-missing`

Runs incremental classification for saved items that are `ready` or `partial` and either have no agent content category or have a stale `contentHash`. The endpoint batches orchestration but intentionally calls the LLM once per item, which keeps result mapping, retries, and future on-device queue behavior simple. Current clients call this automatically after list load/reload when the configured model is reachable, and also expose a manual Analyze action under Search.

`POST /api/agent/items/:id/classify`

Runs the selected local model against one saved item, stores the result, and returns the updated public item.

```ts
type AgentClassification = {
  primaryCategory:
    | "technical"
    | "product"
    | "business"
    | "research"
    | "news"
    | "opinion"
    | "tutorial"
    | "reference"
    | "social"
    | "media"
    | "other";
  contentCategory: {
    id: string;
    label: string;
    description?: string;
    source: "existing" | "new";
  };
  intent: "read_later" | "learn" | "reference" | "follow_up" | "summarize" | "watch" | "share" | "other";
  topics: string[];
  summary: string;
  keyPoints: string[];
  confidence: number;
  language?: string;
  needsFollowUp: boolean;
};
```

Invalid model JSON returns `502`. Unavailable local runtime returns `503`.

The detail panel exposes this as an Agent section with a Classify/Reclassify action. The middle library column shows AI topic filters under Search using `LibraryStats.agentCategories`, and `GET /api/items?agentCategoryId=<id>` filters the list by the generated category. Classification never changes user tags automatically.
